Your configuration file is a JSON file that defines proxy connections (itâ€™s location is `~/.director/config.json`). Here's how to structure it:

```json
{
  "proxies": [
    // Your proxy configurations go here
    {
		  "id": "unique-proxy-name",
		  "name": "Display Name for Proxy",
		  "description": "What this proxy does (optional)",
		  "servers": [
		    // Server connections go here
		  ]
		}
  ]
}

```

# Proxy Configuration

**Required fields:**

- `id` - A unique identifier (no spaces, use dashes or underscores)
- `name` - A friendly name that will be displayed
- `servers` - At least one server connection

**Optional fields:**

- `description` - Help text explaining what this proxy does

# Server Connections

Each proxy can connect to a server in one of two ways:

### Stdio Transport

Use this when your server is a program you run from the command line:

```json
{
  "name": "my-server",
  "transport": {
    "type": "stdio",
    "command": "/path/to/program",
    "args": ["--option", "value"],
    "env": {
      "API_KEY": "your-key-here"
    }
  }
}

```

- `command` - Path to the program to run
- `args` - Command line arguments (optional)
- `env` - Environment variables to set (optional)

### HTTP Transport (Streamable or SSE)

Use this when your server is a web API:

```json
{
  "name": "my-api",
  "transport": {
    "type": "http",
    "url": "https://api.example.com/endpoint"
  }
}

```

- `url` - The web address of your API. This can be either SSE or Streamable URL
